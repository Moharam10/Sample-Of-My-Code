Resto

This is one of the most recent projects i've worked on. Resto is a restaurant template that I've built to apply the skills I've learned in how to automate the workflow while building the front-end functionality of any website and coding in javascript and/or css. Therefore it represents a turning point in the way I approach the front-end in general. Before, I used to use Bootstrap and utilize the already built-in components and utilities that Bootstrap, Semantic UI or any other modern css Frameworks provides. But then, I came to hear about Gulp, and I do remember at that moment how the word sounded so wierd to me not because I didn't know what Gulp as a word means! but mainly because at some moment in my career, (and honestly speaking it's a very short career :)) I thought you can do anything with html javascript and css alone. Of course you can do everything but wouldn't it be awesome if we can use "Classes" in our javascript code or divide our code in many files and then include them in one file?. How about using functions and variables in css?! Wouldn’t it be awesome if you can use a tool that convert all of your css to a compatible cross-browser css. Instead of writing :
transform:scale(1,1);
-ms-transform:scale(1,1);
-moz-transform:scale(1,1);
-webkit-transform:scale(1,1);
-o-transform:scale(1,1);
you focus only on your code and let the tool do its job?. Thanks to gulp we can achieve all of that and much more. Of course we are still writing js and css, but in much more efficient way, that makes our life much easier and makes our code more maintainable and debuggable . Well, with that is said, let’s go back to the moment when I first heard about gulp, as I said the word sounded weird to me but has awaken my curiosity and after I watched the first 5 mins on a YouTube video in which a decent guy was explaining what is gulp and what it is used for, I decided to take my skills to the next step embark on a new journey. Then I built this website in which I applied all the techniques I’ve learned
